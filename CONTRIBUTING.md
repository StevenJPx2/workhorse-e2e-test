# Contributing

## How to File an Issue

Please report bugs and feature requests on our [GitHub issue tracker](https://github.com/StevenJPx2/workhorse-e2e-test/issues). Before opening an issue, search for duplicates. When filing, provide a clear title, detailed description, steps to reproduce, expected vs. actual behavior, and relevant environment information (OS, version, etc.).

## How to Propose a Change

Fork the repository and create a new branch from `main` for your feature or fix. Make focused commits with clear messages. Push your branch to your fork and open a pull request against the upstream repository. In your PR description, explain what problem it solves and reference related issues. Be prepared for feedback and iteration.

## Code of Conduct

We are committed to providing a welcoming and inclusive environment for all contributors. Be respectful of others, avoid discriminatory language, and engage constructively. We do not tolerate harassment, discrimination, or disruptive behavior. Please report violations to the maintainers for prompt resolution.
